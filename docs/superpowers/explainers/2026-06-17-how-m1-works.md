# How M1 (Choice Model) Works

**Question:** How can M1 predict what product a consumer will choose from their CDT embedding?

**Short answer:** M1 is a two-tower neural network that takes a CDT embedding (who they are) and product features (what's available) and outputs the probability of choosing each product.

---

## The Problem: From "Who Is This?" to "What Will They Do?"

### Current State (Prototype)

The fusion meta-learner produces a **CDT embedding** [128] that answers "who is this person?":

- **Identity:** "This is Jane, price-lex quality-seeker"
- **Archetype:** "Compensatory decision strategy"
- **Traits:** "High brand loyalty, medium involvement"

But it **cannot answer** "what will Jane choose when presented with 5 products?"

**Why?** The current generator makes choices randomly:

```python
# generator/trace_simulator.py (line 848)
final_choice = str(rng.choice(alts))  # Uniform random among alternatives
```

The choice is **independent of**:
- What Jane actually inspected in her decision process
- Jane's price sensitivity, brand loyalty, or strategy
- The product attributes Jane saw

This means:
- ❌ No relationship between trace (what Jane inspected) and choice (what Jane chose)
- ❌ CDT embedding (which encodes the trace) cannot predict the choice
- ❌ M1 cannot learn from current synthetic data

---

## The Solution: Preference-Driven Choices

### Phase 0: Fix the Generator

**Step 1: Create Product Schema**
```python
# schemas/product.py
@dataclass
class Product:
    product_id: str          # "prod_electronics_premium_001"
    category: str           # "electronics"
    price_normalised: float # 0.0-1.0 (1.0 = most expensive)
    brand: str              # "Sony", "Samsung", "LG"
    quality_normalised: float # 0.0-1.0 (from ratings)
    features: list[str]     # ["4K", "HDR", "Smart TV"]
```

**Step 2: Create ChoiceSet Schema**
```python
# schemas/choice_set.py
@dataclass
class ChoiceSet:
    choice_set_id: str                    # Links to TrialRecord
    participant_id: str
    n_alternatives: int                    # Number of products shown
    alternative_products: dict[str, str]     # {"A": "prod_xxx", "B": "prod_yyy"}
    displayed_attributes: dict[str, dict[str, float]]  # {"A": {"price": 0.8, "quality": 0.6}}
    chosen_alternative: str                # "B" (what they actually chose)
    choice_probabilities: dict[str, float] # {"A": 0.2, "B": 0.7, ...} (from softmax)
```

**Step 3: Implement Preference-Driven Choice**

Replace random choice with **strategy-based choice** tied to inspected cells:

```python
# generator/trace_simulator.py
def compute_choice_from_inspected_cells(
    inspected_cells: list[dict],  # Cells Jane actually opened
    strategy_params: StrategyParams,  # Jane's decision strategy
    products: dict[str, Product],  # Product attributes
    temperature: float = 0.5,  # Pinned softmax temperature
) -> str:
    """
    Compute choice utility from inspected cells, not full attribute matrix.
    This couples the trace (what was inspected) to the choice (what was chosen).
    """
    utilities = {}
    for slot, product_id in alternative_products.items():
        # Get attribute values for products Jane actually saw
        inspected_values = get_inspected_attributes(slot, inspected_cells)
        
        # Compute utility based on Jane's strategy (lexicographic, compensatory, etc.)
        utility = 0.0
        if strategy_params.rule == "lexicographic":
            # Choose best on first_attribute among inspected cells
            utility = 1.0 if inspected_values[strategy_params.first_attribute] == max_value else 0.0
        elif strategy_params.rule == "compensatory":
            # Weighted sum over inspected attributes
            for attr, weight in strategy_params.attribute_weights.items():
                if attr in inspected_values:
                    utility += weight * inspected_values[attr]
        # Add strategy_lapse noise
        utility += rng.normal(0, strategy_params.strategy_lapse)
        utilities[slot] = utility
    
    # Softmax with pinned temperature
    exp_utils = {slot: np.exp(u / temperature) for slot, u in utilities.items()}
    sum_exp = sum(exp_utils.values())
    probs = {slot: exp_u / sum_exp for slot, exp_u in exp_utils.items()}
    
    # Sample choice
    chosen_slot = rng.choice(list(probs.keys()), p=list(probs.values()))
    return chosen_slot
```

**Key insight:** The choice is now a **deterministic function of (inspected cells, strategy params)**, which means the **trace predicts the choice**.

## M1 Architecture: Two-Tower Choice Model

### The Model

```
Tower 1: Consumer (CDT)
CDT [128] → Linear(128, 64) → ReLU → ┐
                                       ├── Concat [128] → Linear(128, 1) → sigmoid → P(choose)
Tower 2: Product (Features)
Product features [D] → Linear(D, 64) → ReLU → ┘
```

**Where D = number of product attributes** (price, brand, quality, etc.)

### Training Data Format

Each trial produces **N rows** (N alternatives, 1 chosen + N-1 rejected):

```python
# For Jane's trial with 3 products (A, B, C), she chose B:
{
    "participant_id": "compensatory_042",
    "cdt_embedding": [128 floats from frozen fusion],
    "product_features": [price=0.8, quality=0.6, ...],  # Product A
    "chosen": 0  # Not chosen
},
{
    "participant_id": "compensatory_042",
    "cdt_embedding": [128 floats from frozen fusion],
    "product_features": [price=0.5, quality=0.9, ...],  # Product B
    "chosen": 1  # Chosen ✓
},
{
    "participant_id": "compensatory_042",
    "cdt_embedding": [128 floats from frozen fusion],
    "product_features": [price=0.7, quality=0.4, ...],  # Product C
    "chosen": 0  # Not chosen
}
```

### Training Objective

**Binary cross-entropy (pointwise):**
```python
loss = -[chosen * log(P(choose)) + (1-chosen) * log(1-P(choose))]
```

**Future upgrade:** BPR (Bayesian Personalized Ranking) loss for pairwise ranking

## Why This Works: The Critical Insight

### Trace-Choice Coupling

**Before (random choice):**
```
Trace (what Jane inspected) ⊥ Choice (what Jane chose) | Persona
```
The trace and choice are independent. The CDT encoding the trace has **no information** about the choice.

**After (preference-driven choice):**
```
Choice = f(Trace, StrategyParams) | Persona
```
The choice is a **deterministic function of the trace** (via the inspected cells). The CDT encoding the trace now **predicts** the choice.

### Verification: Trace-Only Classifier Check

**Phase 0 Step 10:** Verify trace predicts choice

```python
# Fit logistic regression on trace embedding to predict chosen slot
trace_embedding = trace_encoder.encode(trace)
X = trace_embedding  # [batch, trace_dim]
y = chosen_slot      # [batch]

# Measure accuracy
accuracy = logistic_regression.score(X, y)
chance_baseline = 1 / mean(n_alternatives)  # e.g., 1/3 for 3 alternatives

# Require: accuracy ≥ 1.5 × chance_baseline
assert accuracy >= 1.5 * chance_baseline
```

**If this check passes:** Trace and choice are coupled. M1 can learn from the CDT.

**If this check fails:** Trace and choice are still decoupled. Fix the generator before M1.

## M1 Success Criteria

### 1. CDT Contribution (The Gating Criterion) ⭐

**Requirement:** M1 must beat a **no-CDT baseline** by **≥ 0.05 AUC**

**No-CDT baselines (same two-tower, different tower 1):**
1. **Product-features-only:** Tower 1 = product features (no consumer info)
2. **Persona-id one-hot:** Tower 1 = one-hot persona (demographics only)

**Why this matters:** Proves the *decision process* (trace-dominated CDT) predicts the choice, not just product features or persona identity.

**If lift < 0.05:** M1 fails, even if absolute AUC is high. Likely symptom of trace–choice decoupling.

### 2. Discrimination (Floor)

**Requirement:** AUC ≥ 0.65 on held-out participants

**Subordinate to lift gate:** This is a minimum, but passing lift is more important.

### 3. Calibration (Recovery Check)

**Requirement:** Brier ≤ 0.25 and calibration slope ∈ [0.8, 1.2]

**Note:** This verifies *recovery of the known generative process*, not real-world calibration. Labels come from the generator's known parameters.

   class ChoiceModel(nn.Module):
       def __init__(self, cdt_dim=128, product_dim=D, hidden=64):
           self.consumer_tower = nn.Sequential(
               nn.Linear(cdt_dim, hidden),
               nn.ReLU(),
           )
           self.product_tower = nn.Sequential(
               nn.Linear(product_dim, hidden),
               nn.ReLU(),
